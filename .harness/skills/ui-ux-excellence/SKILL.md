# UI/UX 品质原则

## 一眼即懂

追求的效果：打开画面的一瞬间，不需要阅读说明书就知道能做什么。最重要的信息在最显眼的位置。操作的优先级是视觉呈现的——主要按钮比次要按钮更大或更亮。状态信息（回合、阶段、存活人数）在固定的位置，余光就能扫到。危险操作有二次确认——但不是烦人的弹窗，而是自然的"再点一次"。

避免的效果：需要找半天才能操作、所有按钮一样大一样亮分不清主次、信息散落各处要来回扫视。

## 信息架构

追求的效果：信息不是平铺的——有层级。最上层是"现在该干什么"，下一层是"当前状态"，再下一层是"详细数据"。玩家可以深入查看细节，但不强制。复杂信息（12个玩家的状态）用视觉编码——颜色、图标、位置——而不是文字堆砌。同类型信息在一起，不同类型之间有明确分隔。

避免的效果：信息过载让人不想看、或者信息太少让人猜、重要数据藏在深处要点好几下。

## 操作直觉

追求的效果：常用操作不超过两次点击。相关的操作放在手指自然到达的位置。按钮的大小和间距适合鼠标点击——不会误触相邻按钮。拖拽、滚轮、右键有合理的行为，不仅仅依赖左键。键盘快捷键覆盖核心操作，且在界面有提示。

避免的效果：关键操作藏很深、按钮太小容易点错、只有鼠标能用键盘完全无效。

## 状态可见性

追求的效果：系统当前状态始终可见。加载中、等待中、处理中有明确指示。错误有清晰的解释和修复建议。成功有确认但不打扰。长时间操作有进度或预估时间。游戏阶段切换时状态栏同步更新，不存在"显示的是旧数据"的情况。

避免的效果：点完不知道发生了什么、错误信息看不懂、进度条卡住不动。

## 适应性与包容性

追求的效果：界面在不同分辨率下保持可用的布局——不只是缩放，是重新排列。文字大小可读，对比度足够——深色背景上的浅色文字，或浅色背景上的深色文字，没有"眯眼才能看清"的情况。色盲友好的配色——不用仅靠红绿区分重要信息。窗口化、全屏、不同宽高比都能正常显示核心功能。

避免的效果：缩小窗口后按钮被裁掉、低对比度文字看不清、只有红绿色区分导致色盲无法使用。

## 自主权

以上是体验目标。用什么控件、怎么布局、什么颜色、什么交互方式——由你决策。只要最终让玩家感到"好用"而不是"能用"。
